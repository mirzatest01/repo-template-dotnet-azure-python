# Security Policy

## Supported Repositories
This security policy applies to all repositories within this organization
unless a repository defines a more specific policy.

---

## Reporting a Vulnerability
If you discover a security vulnerability, please do NOT open a public issue.

Instead:
- Email: security@your-org.com
- Include:
  - A detailed description of the issue
  - Steps to reproduce
  - Affected components or repositories
  - Any proof-of-concept or logs (if available)

---

## Response Timeline
We aim to:
- Acknowledge receipt within 2 business days
- Provide an initial assessment within 5 business days
- Communicate remediation or mitigation plans as appropriate

Timelines may vary depending on severity and impact.

---

## Disclosure Policy
- Please do not publicly disclose vulnerabilities until remediation is complete
- Coordinated disclosure will be handled in collaboration with the reporter
- Credit may be given to reporters unless anonymity is requested

---

## Security Best Practices
All contributors are expected to:
- Avoid committing secrets or credentials
- Use approved dependencies and libraries
- Follow secure coding practices
- Respect data privacy and compliance requirements

---

## AI and Security
AI-generated code must:
- Be reviewed for security vulnerabilities
- Not introduce insecure defaults
- Follow organizational security standards
